** Using

*** Build and run

Build and run via the [[file:justfile][justfile]]:

#+begin_src bash
  just            # list recipes
  just build      # build the docker image
  just run        # run with ./etc-asterisk bind-mounted into the container
  just lint       # hadolint the Dockerfile
  just config-test # boot Asterisk briefly and fail on ERROR lines
  just push       # push to the registry
  just all        # build + push
#+end_src

Alternatively use docker compose (adds persistent volumes for
=/var/lib/asterisk= and =/var/log/asterisk=):

#+begin_src bash
  docker compose up --build
#+end_src

*** TLS certificates

The SIP-TLS transport (port 5061, configured in
=etc-asterisk/pjsip.conf= under =[transport-tls]=) expects a cert and
key under =etc-asterisk/keys/=. Generate a fresh self-signed pair with:

#+begin_src bash
  just gen-keys                       # CN=asterisk.local, valid 10 years
  just gen-keys pbx.example.lan 365   # custom CN and validity (days)
#+end_src

This writes two files into =etc-asterisk/keys/= (both gitignored):

| File           | Role             | Where it lives                                |
|----------------+------------------+-----------------------------------------------|
| =asterisk.key= | Private key      | Server only — never copy off the box          |
| =asterisk.pem= | Public cert      | Distribute to TLS clients that need to trust it |

A self-signed cert is not in any client's default trust store, so each
SIP-TLS client must either:

1. Import =asterisk.pem= as a trusted root, *or*
2. Disable certificate verification (fine for a homelab; not for prod).

In pjsua, =--use-tls= turns on TLS; trust-store flags vary by build —
see =pjsua --help= for =--ca-list=.

*** Asterisk CLI commands

Inside the running Asterisk console:

#+begin_src bash
  # Turn on pjsip logger
  pjsip set logger on
  # Show registrations
  pjsip show registrations
  # RTP show settings
  rtp show settings
  pjsip show aors
  pjsip show contacts
  core show channels

  # list available codecs
  core show codecs
#+end_src

** Hardware (FXS / ATA configuration)

The =[ata]= endpoint template in =pjsip.conf= covers analogue phones
behind a SIP-to-FXS adapter. Plug a regular analogue phone into the
adapter's =FXS Phone= jack with an RJ-11 cable; the adapter handles
the SIP side.

Extension range for ATAs is =6010-6019= (see =pjsip.conf=). Each FXS
port on a multi-port adapter registers as its own extension.

*** Common settings (same across all three devices)

| Setting              | Value                                |
|----------------------+--------------------------------------|
| SIP server / proxy   | =$PBX= (the Asterisk host)           |
| SIP port             | =5060=                               |
| Transport            | UDP                                  |
| Outbound proxy       | (leave blank)                        |
| User / Auth ID       | =6010=, =6011=, … (one per FXS port) |
| Password             | matching =pjsip.conf=                |
| Register expires     | 300 s (5 min)                        |
| Preferred codecs     | G.711 µ-law (=PCMU=) then A-law (=PCMA=) |
| DTMF mode            | RFC2833                              |
| NAT traversal        | off (LAN — set =Keep-Alive= if remote) |
| Use DNS SRV          | no                                   |

Our =[ata]= template restricts the negotiated codec list to ulaw/alaw,
so picking anything else on the adapter just shortens the SDP offer
and isn't useful.

*** Grandstream HT812

Two FXS ports → two extensions. Default web UI at =http://<device-ip>=,
login =admin / admin= (older units) or =admin / <random-on-sticker>=
(newer factory-default). The HT812 has a per-port =FXS PORT1= / =FXS PORT2=
config page and the relevant fields to set on each:

| HT812 field                | Value                          |
|----------------------------+--------------------------------|
| Account Active             | Yes                            |
| Primary SIP Server         | =$PBX=                         |
| SIP Transport              | UDP                            |
| SIP User ID                | =6010= (port 1), =6011= (port 2) |
| Authenticate ID            | same as SIP User ID            |
| Authenticate Password      | matching =pjsip.conf=          |
| Name (display)             | e.g. "Hallway phone"           |
| Preferred Vocoder choice 1 | PCMU                           |
| Preferred Vocoder choice 2 | PCMA                           |
| Send DTMF                  | via RTP (RFC2833)              |
| NAT Traversal              | No (LAN)                       |
| Register Expiration        | 5 (minutes)                    |
| Use Random SIP Port        | No                             |

After saving, the =Status= page should show =Registered= for the port
you configured. Cross-check on the PBX:

#+begin_src bash
  docker exec -it $(docker ps -qf ancestor=sm6wjm/asterisk) \
      asterisk -rx "pjsip show endpoint 6010"
#+end_src

The contact line should list the HT812's LAN IP.

*** Cisco SPA122

Two FXS ports → two lines. Web UI =http://<device-ip>=, default login
=admin / admin=. Lineage: Sipura → Linksys → Cisco, so the UI feels
dated. Two config pages — =Voice → Line 1= and =Voice → Line 2=:

| SPA122 field          | Value                            |
|-----------------------+----------------------------------|
| Line Enable           | yes                              |
| Proxy                 | =$PBX=                           |
| Outbound Proxy        | (blank)                          |
| Use Outbound Proxy    | no                               |
| Register              | yes                              |
| Register Expires      | 3600                             |
| SIP Port              | 5060                             |
| SIP Transport         | UDP                              |
| User ID               | =6010= (line 1), =6011= (line 2) |
| Password              | matching =pjsip.conf=            |
| Auth ID               | same as User ID                  |
| Use Auth ID           | yes                              |
| Preferred Codec       | G711u                            |
| Use Pref Codec Only   | no (allow fallback to alaw)      |
| DTMF Tx Method        | AVT (= RFC2833)                  |

Reboot after saving (Cisco's web UI is cranky about applying changes
without one). The =Info= page shows registration status per line.

*** Cisco ATA 191

Single FXS port. Successor to the SPA112. Web UI =http://<device-ip>=,
default login =admin / admin= or =cisco / cisco= depending on firmware.

| ATA 191 field         | Value                            |
|-----------------------+----------------------------------|
| Voice → SIP → Proxy 1 | =$PBX=                           |
| Voice → SIP → Port 1  | 5060                             |
| Voice → Line 1 Enable | yes                              |
| User ID               | =6010=                           |
| Password              | matching =pjsip.conf=            |
| Auth ID               | same as User ID                  |
| Preferred Codec       | G.711u                           |
| DTMF Tx Method        | AVT                              |
| Register Expires      | 300                              |

Save & reboot. The =System Status= page lists each line's registration
state.

Reference: https://docs.pjsip.org/en/latest/specific-guides/other/cli_cmd.html

*** Building pjproject

#+begin_src bash
  sudo apt install build-essential libasound2-dev libssl-dev libncurses5-dev \
       libreadline-dev libavcodec-dev libavformat-dev libswscale-dev libopus-dev

  git clone https://github.com/pjsip/pjproject.git
  cd pjproject
  ./configure --enable-shared
  make dep
  make -j$(nproc)
#+end_src

The compiled apps land in =pjproject/pjsip-apps/bin/=.

*** Running pjsua

=PBX= points at the Asterisk host. Default =127.0.0.1= assumes pjsua
runs on the same machine as the PBX; override for other hosts, e.g.
=PBX=192.168.1.16= or =PBX=asterisk.local=.

To actually call between two extensions you need **two pjsua processes
running side by side**, one registered as each extension. They use
different =--local-port= values so they don't fight for the same UDP
port. Open *two terminals* and run one block in each:

Terminal 1 — register as 6002:

#+begin_src bash
  PBX=${PBX:-127.0.0.1}
  pjsua --id sip:6002@$PBX \
        --registrar sip:$PBX \
        --realm '*' \
        --username 6002 \
        --password 6002 \
        --local-port=5063
        # --use-tls   # add for SIP-TLS on 5061
#+end_src

Terminal 2 — register as 6001:

#+begin_src bash
  PBX=${PBX:-127.0.0.1}
  pjsua --id sip:6001@$PBX \
        --registrar sip:$PBX \
        --realm '*' \
        --username 6001 \
        --password 6001 \
        --local-port=5062
#+end_src

Verify both are registered (in a third terminal, against the running
Asterisk container) — both endpoints should show =Available= with a
contact listed:

#+begin_src bash
  docker exec -it $(docker ps -qf ancestor=sm6wjm/asterisk) \
      asterisk -rx "pjsip show endpoints"
#+end_src

*** Useful pjsua commands

Once pjsua is running you get an interactive prompt. The most useful
single-letter commands:

| Key  | What it does                                                   |
|------+----------------------------------------------------------------|
| =m=  | Make a call (prompts for the destination URI)                  |
| =a=  | Answer the incoming call with 200 OK                           |
| =g=  | Answer with a specific code (=200= answer, =486= busy, =603= decline) |
| =h=  | Hang up the current call (=ha= hangs up all)                   |
| =H=  | Hold the call                                                  |
| =v=  | re-inVite — release hold / resume                              |
| =]= =[= | Switch to next / previous call (when more than one is active) |
| =x=  | Blind transfer the current call                                |
| =#=  | Send DTMF (e.g. to enter a voicemail PIN, or dial =*97=, =*43=) |
| =dq= | Dump current call quality (jitter, loss, RTT) — best audio-debug tool |
| =d=  | Dump status (accounts, registrations, active calls)            |
| =Cp= | Show / change codec priorities                                 |
| =V=  | Adjust audio volume                                            |
| =rr= | Re-register the account (useful when toggling network)         |
| =q=  | Quit                                                           |

The full menu is printed any time you press =?= at the prompt.

*** Making calls

Press =m=, then type the destination URI at the =Make call:= prompt.
This is pjsua's own CLI — *not* a shell — so the =$PBX= variable used
above is NOT expanded here. Substitute the actual address (or
=127.0.0.1= when pjsua is on the same machine as Asterisk):

#+begin_example
  sip:6001@127.0.0.1;transport=udp     ; another softphone
  sip:6010@127.0.0.1;transport=udp     ; an ATA line
  sip:*43@127.0.0.1;transport=udp      ; echo test — repeat your voice
  sip:*44@127.0.0.1;transport=udp      ; music on hold — one-client audio test
  sip:*97@127.0.0.1;transport=udp      ; check your own voicemail
#+end_example

(If you set =PBX=192.168.1.16= when starting pjsua, use that IP in the
URIs above. The variable substitution happens only in the shell command
that launches pjsua, not inside pjsua's prompt.)

To answer an incoming call, press =a=. To reject with a specific status,
press =g= and pick:

#+begin_example
  200 → answer
  486 → busy
  603 → decline
#+end_example

*** Things to try

- =*43= from any phone — the echo test will repeat your voice back. If
  you hear yourself, the audio path is healthy in both directions.
- Park / unpark via =x= (transfer) to a parking lot extension (not
  configured by default — would need =res_parking.so= loaded and a
  =parkext= in =features.conf=).
- =dq= during a call: watch the jitter and packet-loss counters when
  moving the mobile client between WiFi and cellular.
- Change codec priority with =Cp= and observe negotiation in the
  Asterisk CLI with =pjsip set logger on= → look for the m= line in
  the SDP.
